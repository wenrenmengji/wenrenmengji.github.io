Git

对于 Git 我想说的是在这个月我看完了《Pro Git》这本书，学到了关于 Git 的相关命令，笔记放在了软件 Notion 里。现在想想，当时对 Git 非常感兴趣，在 Google 中找到了开源仓库，并且找到了这本书的作者 Scott Chacon （斯科特·查孔）以及他的博客，非常奇妙。
当时使用 Git 时，我将 master 修改成了 main 。有两种方法：一种是直接编辑 config 文件：

验证：git config --global init.defaultBranch，返回“main”
另一种是在 cmd 运行git config --global init.defaultBranch main，验证同上。

与命令行 CLI 相关的有这些：

  • 上下箭头切换历史命令
  • pwd 显示当前目录
  • ls 输出当前目录的所以文件
  • Git 分页器，按 Q 退出分页器
  • 切换目录主要使用 cd 命令。cd <目标路径>-切换目录。cd . -显示当前目录 。cd ~ -用户主目录: /c/Users/21199

问题：

  • 在 git bash 第一行显示的是“21199@▒▒▒ŵ▒▒ڱ▒ MINGW64 ~/Desktop”，部分内容看不清楚——解决：检查系统区域设置（Windows）打开 控制面板 → 区域 → 管理 → 更改系统区域设置。勾选 Beta: 使用 Unicode UTF-8 提供全球语言支持 → 重启电脑。
  • Git 命令中包含 {},出现错误,导致命令运行不了。——解决：将 {} 加引号。

cmd

与命令行
  • C:/D:/E:（切换磁盘
  • md xxx（创建文件夹 mkdir（创建目录
  • cd xxx（切换文件夹
  • cd ..（返回上一级目录。可以嵌套如../../..
  • cd ../../../../..（返回多级目录。（方法二-绝对路径跳转：cd
  • D：\目标路径
  • cd -（快速返回上一个目录
  • cd（直接退回到用户主目录
  • rd（rmdir xxx（删除文件夹
  • rd /s /q "父文件夹\子文件夹"（递归删除子文件夹。/s（递归删除。/q（静默删除，不提示确认。（删除文件夹里的文件夹）
  • cd（chdir或者echo %cd%（显示当前目录
  • 上下箭头切换历史命令
  • 在输入不完整的文件名后敲击Tab键自动补全（目录的制表符补全 dir（显示当前目录的所有文件
  • code .（用vscode打开当前目录
  • chcp（编码方式 。输出示“Active code page：65001”
  • taskkill /IM "程序名.exe" /F（关闭进程
  • ping
  • break > filename.txt（新建文件, 但不能创建以 . 开头的文件。——在git环境中, 可以用 ni .gitignore 创建 .gitignore文件

code

操作:

  • Ctrl + C（调出集成终端——Ctrl+Shift+~ 创建多个终端
  • Ctrl + F（单个文件搜索，匹配到的结果高亮显示——Ctrl + Shift + F 全局搜索

Code 内置的 Git 状态标识功能

  • yellow M（modified 已修改未暂存
  • green M（已修改且已暂存
  • green U（新增未跟踪 untracked
  • 源代码管理器（Ctrl+Shift+G
    问题：
  • code里的终端为何是Windows操作系统里的终端？——vscode终端实际上是一个”嵌入式终端模拟器“，它直接调用系统中默认配置的命令行工具，而不是自己实现一个独立的终端。牵扯系统集成、环境变量继承、一致性。
  • 在vscod中运行hexo服务器之后，就不能在终端输入其他命令。——“Ctrl+SHift+ ` “（新建一个终端窗口。（原终端可以继续运行“hexo server，新终端可以自由输入其他的command”）、另一种方法是Start-Process -NoNewWindow hexo server ”/“hexo server &（后台运行hexo服务器。(关闭Code 后后台进程可能不会自动终止，需手动结束。)

Hexo

命令
  • hexo new （layout）（titlt）
    • hexo new page --path about/me "about" #以上命令会创建一个 source/about/me.md 文件，同时 Front Matter 中的 title 为 "About me"， title 是必须指定的！！！
    • hexo new "文章" #会在 source/_post/文章.md 文件
  • hexo generate
    • #生成静态文件。
  • hexo deploy
    • #部署网站，在此之前需要先运行 hexo generate 后生成静态文件。
  • hexo clean
    • #清除缓存文件 (db.json) 和已生成的静态文件(public) 。
